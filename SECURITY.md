# Security Policy

**Languages:** [English](./SECURITY.md) · [Español](./docs/es/SECURITY.md)

## Supported versions

Simple Template is pre-1.0. Only the latest minor version receives security fixes.

| Version | Supported |
|---------|-----------|
| 0.1.x   | Yes       |
| < 0.1   | No        |

## Reporting a vulnerability

Please **do not** open a public issue for security reports. Instead, contact the maintainer directly at:

**jesus.cocano@gmail.com**

When reporting, include:

- A clear description of the issue and its impact
- Steps to reproduce (proof-of-concept if possible)
- The affected version(s) and platform(s)
- Whether the vulnerability has been disclosed elsewhere

## Response timeline

- **Acknowledgement:** within 72 hours
- **Initial assessment:** within 7 days
- **Fix or mitigation plan:** communicated as soon as feasible, depending on severity

Because this project is maintained by a single person, response times may vary. Thanks for your patience.

## Disclosure policy

The maintainer follows a coordinated disclosure approach:

1. You report the issue privately.
2. We confirm and investigate.
3. A fix is prepared and released.
4. Once users have had reasonable time to update, details are published in the changelog and, if warranted, a GitHub Security Advisory.

Credit is given to the reporter unless they request anonymity.

## Scope

This policy covers the Simple Template application code hosted in this repository. It does not cover:

- Third-party dependencies — report those upstream to the respective maintainers
- Issues in services you connect Simple Template to (your SMTP provider, your CDN, etc.)

## Thank you

Responsible disclosure makes open source safer for everyone. Thanks for helping protect the Simple Template community.
